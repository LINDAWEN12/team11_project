2.AI对话工具选择
1. Claude（Anthropic）
核心优势：目前长文档理解和生成质量最突出的工具。上下文窗口大且计价统一，特别适合直接上传完整的项目需求书、技术规格书，能精准提取核心矛盾和逻辑漏洞。生成的文档结构严谨，语气专业。

主要短板：免费版额度严格，非订阅用户高峰期排队明显；代码生成能力略弱于 ChatGPT。

成本：Team 版约 ¥175/月，API 调用 Sonnet 4.6 性价比在同类中较优。

2. 文心一言（百度）
核心优势：中文语义理解和表达最自然，对本土行业术语、政策背景的生成质量高。完全免费，支持上传较长的中文 PDF 并直接全文翻译或摘要。

主要短板：复杂逻辑推理和多轮需求变更后的上下文连贯性稍弱于上述两款海外产品；API 调用稳定性偶有波动。

成本：网页和 App 端免费。




3.AI审查工具选择
候选 1：CodeRabbit
评估记录
CodeRabbit 是当前 AI 代码审查领域市场占有率最高的工具，已处理超过 1300 万个 PR，连接超过 200 万个仓库。其技术方案结合 LLM 推理与 40 多种静态分析工具，在独立基准测试中达到 46% 的真实运行时 Bug 检测率，精度为 95.88%。对课程项目而言，最直接的优势在于 GitHub App 一键安装后无需任何配置即可开始使用，每次 PR 自动生成审查摘要和逐行评论，极大降低了工具使用门槛。
选择理由
易用性：GitHub App 一键安装，零配置启动，适合课程项目短周期特点。
成本：个人开发者免费版支持无限仓库，完全覆盖课程项目需求。
功能：PR 自动摘要和行级评论，反馈直观，无需额外查看仪表盘。
学习价值：每一条 AI 建议都附带解释，有助于提升代码质量意识。
候选 2：SonarQube (Community Edition)
评估记录
SonarQube 是代码质量分析领域最成熟的工具之一，社区版完全免费且开源。2025 年后 SonarQube 引入了 AI CodeFix 功能，可为检测到的问题提供一键修复建议。特别值得注意的是，SonarQube 具备自动检测 GitHub Copilot 生成代码的能力，并可通过 AI Code Assurance 工作流进行专门审查——这与本课程的《AI 协作契约》目标高度契合。部署方式上，社区版可通过 Docker Compose 在本地运行，与 AI 编程助手形成“AI 写代码 → SonarQube 扫描 → AI 修复”的闭环工作流。
选择理由
成本：社区版完全免费开源，无任何使用限制。
与课程目标契合：内置 AI 生成代码检测功能，直接服务于“AI 使用追溯”这一课程要求。
可自托管：本地 Docker 部署，不依赖外部服务，符合数据隐私要求。
CI 集成成熟：GitHub Actions 官方 Action 支持，与任务四的 CI 配置无缝衔接。


4.AI测试工具选择
1.通义灵码。选择理由：功能性强，能较好地实现生成测试用例的任务；可免费使用；作为IDE内置插件，使用易上手。
2.APIfox。选择理由：可以实现AI生成接口测试用例的功能；个人版可免费使用；导入API文档后即可触发 AI 生成，无需复杂配置，操作简单。
