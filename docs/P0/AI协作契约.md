第一条：AI 使用原则
- AI 是工具，工程师是责任主体
- 所有 AI 输出必须经过人工审查后才能合入项目

第二条：AI 使用范围
- 允许使用 AI 的场景：
辅助编写项目文档（README、API 接口说明）的初稿。
生成单元测试的样板代码、Mock 数据构造。
解释不熟悉的第三方库函数用法。
辅助 Debug（粘贴报错日志寻找解决思路）。
格式化数据（如 JSON 转结构体定义）。

- 禁止直接使用 AI 输出（需额外审查）的场景：
核心业务逻辑代码（如订单金额计算、权限校验逻辑）。
安全相关代码（如密码加密、SQL 语句拼接、JWT 生成逻辑）。
涉及架构决策的配置（如数据库连接池参数、缓存策略）。

- 完全禁止使用 AI 的场景：
伪造测试数据中的个人隐私信息。
生成包含公司/项目内部密钥、Token、密码的配置文件。

第三条：过程记录要求
- 所有与 AI 的交互 Prompt 必须保留日志
- Git commit message 必须标注 [AI-assisted] 或 [Human-written]
- 每阶段填写《AI 协作反思日志》

第四条：代码合入规则
- AI 生成的代码必须通过至少一名团队成员的 Code Review
- 每位成员每个编码阶段至少有 2 个完全手写的核心函数
